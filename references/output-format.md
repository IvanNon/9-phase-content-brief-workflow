# Output Format

The final brief structure produced by Phase 9. Two variants: standard (12 sections) for non-YMYL content, extended (14 sections) for YMYL content.

The brief is delivered as inline markdown by default. If the user requests Excel or Word, generate the same content structure in the requested format.

---

## Section structure overview

### Standard format (non-YMYL): 12 sections

1. Project Specifications
2. Technical Requirements & Quality Standards
3. SEO Strategy & Keyword Targeting
4. Query Fan-Out Content Enhancement Strategy
5. Article Structure & Content Architecture
6. Semantic Keyword Integration Strategy
7. Strategic Internal Links
8. External Citations & Authority References
9. Brand/Product Integration
10. Copywriter Tone Directive
11. Comprehensive FAQ Section
12. Final Validation Checklist

### Extended format (YMYL): 14 sections

1. Project Specifications
2. Technical Requirements & Quality Standards
3. SEO Strategy & Keyword Targeting
4. Query Fan-Out Content Enhancement Strategy
5. Article Structure & Content Architecture
6. Semantic Keyword Integration Strategy
7. Strategic Internal Links
8. **Scientific Citations & Research Foundation** (replaces Section 8)
9. **Expert Quotes & Authority Validation** (new for YMYL)
10. Brand/Product Integration
11. Copywriter Tone Directive
12. **E-E-A-T Compliance Framework** (new for YMYL)
13. **YMYL Content Standards** (new for YMYL)
14. Comprehensive FAQ Section
15. Final Validation Checklist

(Note: numbering shifts because YMYL adds two sections; the example shows 15-section count when both YMYL additions are present, but the conceptual structure is 14 distinct section types.)

## Section-by-section template

### Section 1 — Project Specifications

```markdown
## 1. Project Specifications

**Primary keyword:** [keyword]

**Topic / content angle:** [description]

**Target word count:** [N] words ([content type])

**Publishing surface:** [blog / hub page / knowledge base]

**Conversion goal:** [specific action]

**Success metrics:**
- Ranking: [target position for primary keyword]
- Traffic: [target if applicable]
- Engagement: [target if applicable]
- Conversion: [target rate or volume]

**Content type:** [Long-form pillar / Mid-form article / Short-form post]

**YMYL status:** [Yes / No] — [brief justification]
```

### Section 2 — Technical Requirements & Quality Standards

```markdown
## 2. Technical Requirements & Quality Standards

**Reading level target:** [Flesch-Kincaid grade level, e.g., 8th-10th grade]

**Sentence length norm:** [average sentence length target]

**Paragraph length norm:** [typical paragraph length]

**Mobile optimization:** [scannable structure, short paragraphs for mobile reading]

**Visual element requirement:** 1 visual per ~[N] words ([N] total visuals planned)

**Accessibility:** [alt text on all images, semantic heading hierarchy, descriptive link text]

**Schema markup:** [Article / NewsArticle / HowTo / FAQPage / etc.]

**Meta description target:** [character range, e.g., 150-160 chars]

**URL slug recommendation:** [suggested slug]
```

```markdown
| Section | Target word count | Target mentions | Target density |
|---------|-------------------|-----------------|----------------|
| Introduction | [N] | [N] | [N]% |
| H2 Section 1 | [N] | [N] | [N]% |
```

### Section 13 — YMYL Content Standards (YMYL only)

```markdown

> "[Drafted disclaimer specific to vertical and content]"

- Warnings before suggestions in [specific safety-relevant areas]
```

## Format export options

### Markdown (default)

The brief renders inline as markdown. This is the default behavior.

### Excel workbook

If the user requests Excel:
- Tab 1: Summary (Project Specs, SEO Strategy, Word Count Distribution)
- Tab 2: Content Architecture (Section-by-section structure)
- Tab 3: Keywords (cluster table, density allocation, distribution)
- Tab 4: Citations (internal links + external citations + expert quotes)
- Tab 5: Validation Checklist

Color palette: navy headers (#1F2A44), Calibri throughout, alternating row fill (#F4F5F7), colored severity-style badges if priority levels apply.

### Word document

If the user requests Word:
- Single document with all sections
- Headings styled (H1, H2, H3 with consistent typography)
- Tables formatted with borders
- Print-friendly margins

Use the same color palette as Excel for any colored elements.

## Filename conventions

When generating Excel or Word files:

```
content-brief-{primary-keyword-slug}-{date}.{xlsx|docx}
```

Where:
- `primary-keyword-slug`: lowercase, hyphenated version of primary keyword
- `date`: YYYY-MM-DD

Examples:
- `content-brief-best-running-shoes-plantar-fasciitis-2026-05-03.xlsx`
- `content-brief-how-to-file-tax-extension-2026-05-03.docx`

Save to `/mnt/user-data/outputs/{filename}`. After saving, use `present_files` to share with the user.
